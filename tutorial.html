家宽 SOCKS5 出口网关部署教程

方案目标

公网客户端连接 VPS 的本地端口；AnyTLS 与 NaiveProxy 访问外部网站时，实际出口走另一台家宽服务器提供的 SOCKS5。

公网客户端 → socat 入口容器 → tun2socks 网络命名空间 → 家宽 SOCKS5 → 目标网站

为什么能避免非对称路由

公网入口只进入 socat 容器，socat 再连接 Docker bridge 内固定 IP。tun2socks 容器内为 Docker 入口子网保留 eth0 直连路由，所以业务容器给 socat 的回包仍走 eth0 返回，不会被默认 TUN 路由送往 SOCKS5。脚本还会补齐本项目 Docker 入口网络的专属 NAT/FORWARD 规则，确保自定义 bridge 容器可以连接上游 SOCKS5。

一键执行

INSTALLER_DIR=/tmp/jiakuan-tun2socks-gateway-installer
rm -rf "$INSTALLER_DIR"
git clone https://github.com/jiasongji/jiakuan-tun2socks-gateway.git "$INSTALLER_DIR"
bash "$INSTALLER_DIR/bootstrap-install.sh"

脚本会先询问域名，再默认给出 /www/wwwroot/<域名> 作为项目根目录；也可以输入任意绝对路径。

默认端口

服务宿主机端口入口容器后端目标

NaiveProxy HTTP33802JiaKuan-Entry-NaiveHTTP172.31.253.10:33802

验证命令

bash <项目根目录>/jiakuan-proxy/scripts/verify.sh

docker run --rm --network container:JiaKuan-Tun2Socks jiasongji/jiakuan-curl:8.10.1 -4fsS https://api.ipify.org

回滚命令

bash <项目根目录>/jiakuan-proxy/scripts/rollback.sh

风险提示

  • 脚本不会默认迁移 Docker 全局 data-root；如选择迁移，必须二次输入 MIGRATE。
  • 真实密码写入 jiakuan.env，权限为 600，不提交到 GitHub。
  • 若上游 SOCKS5 不支持 UDP，DNS 或 UDP 业务可能不可用；本方案优先保证 TCP 出口。
  • 脚本会添加仅匹配本项目 Docker 入口子网的 NAT/FORWARD 规则，回滚脚本会删除这些规则。
